# 自动化日报全流程文档

## 任务概述

每天早上9点自动执行资讯同步，输出到 Discord、飞书、Notion

## 执行流程

### Step 1: RSS抓取
- 脚本位置: `~/.openclaw/workspace-dev/scripts/daily_news_sync_v2.py`
- **必须用 XML 解析器**，不是正则！

### Step 2: 日期过滤
- **只取昨日内容**（今日9点同步昨日资讯）

### Step 3: 关键词分类
```
营销+AI: ai/gpt/openclaw/kimi/大模型/agent/智能/机器人/nano banana/gemini
京东: 含"京东"
抖音: 抖音/字节/tiktok
阿里妈妈: 阿里妈妈/万堂书院
小红书: 含"小红书"
电商零售: 电商/零售/店铺/销量/直播带货/爆卖/gmv/商家/外卖/拼多多/天猫/淘宝
营销增长: 营销/增长/投放/广告/案例/品牌/趋势/消费者
```

### Step 4: 输出
- 按昨日实际数量输出，不硬凑
- 分类: 营销+AI、电商零售、营销增长、小红书、京东、抖音、阿里妈妈
- 格式: emoji分类 + 标题 + 原文链接

### Step 5: 飞书文档
- 标题: `每日精选资讯 | YYYY年MM月DD日`
- 每条: 标题 + 链接（分开两行）
- docx 创建端点：`/open-apis/docx/v1/documents`
- 正文写入端点：`/open-apis/docx/v1/documents/{document_id}/blocks/{block_id}/children`
- 纯文本块必须使用 `text.elements[].text_run.content`
- **不要**把文本块写成 `paragraph`
- 给 Gary 授权时：**不要用** `drive/v1/permissions/{token}/members/batch_create`
- 给 Gary 授权时：使用单条 `drive/v1/permissions/{token}/members` create，授予 `full_access`
- Gary openid: `ou_d635f4f3d20ac474cf8575038b5d2b33`
- 如脚本重跑拿不到历史内容，可优先从 agent session JSONL 回捞最近一次成功生成的 `FEISHU_CONTENT`

## 执行命令
```bash
python3 ~/.openclaw/workspace-dev/scripts/daily_news_sync_v2.py
```

## 定时任务
```
0 9 * * * python3 ~/.openclaw/workspace-dev/scripts/daily_news_sync_v2.py
```

## 关键注意事项

1. **RSS解析必须用XML解析器**，不能用正则（2026-03-10的教训）
2. **日期过滤要严格**，只取昨日内容
3. **分类按关键词**，不是按公众号
4. **飞书 docx 写入要走新版 children 接口**，不要再用旧路径
5. **文本块字段必须是 `text`**，不要误写成 `paragraph`
6. **给 Gary 开 full_access 时优先走单条 create**，不要走 `batch_create`
7. **如果当天重跑脚本出现 0 条或结果漂移**，优先从最近一次成功运行日志/JSONL 回捞 `FEISHU_CONTENT`
