### 🎯 深度数据抓取技术

#### 1. JavaScript注入抓取

- ✅ 直接访问页面DOM

### 🔍 数据提取详解

#### 笔记ID提取
```python
import re
# 从 /explore/69132081000000000700831b 提取
match = re.search(r'/explore/([a-zA-Z0-9]+)', url)
```

#### 作者信息提取
- **昵称**：直接从DOM读取
- **ID**：从主页链接提取
- **主页**：`https://www.xiaohongshu.com/user/profile/xxxxx`
- **粉丝数**：悬停或直接读取（支持"万"、"k"格式）

#### 互动数据提取
```python
# 自动识别互动按钮
interactEls = document.querySelectorAll('[class*="interact"] span')

# 根据class名称分类
if (className.includes('like')) → 点赞数
if (className.includes('collect')) → 收藏数
if (className.includes('comment')) → 评论数
if (className.includes('share')) → 分享数
```

#### 内容提取
- **正文**：从 `.note-content` 或 `#detail-desc` 提取
- **长度限制**：500字（防止Excel过大）
- **保留格式**：保留换行和基本格式

#### 媒体链接提取
- **封面链接**：`img[src*="sns-img"]` 的 src 属性
- **视频链接**：`video source` 或 `video` 的 src 属性
- **完整URL**：包含协议和域名

### 🔍 故障排查

#### Q1: 粉丝数为空
**可能原因：**
- 作者隐藏了粉丝数
- 页面加载不完整
- 选择器失效

**解决方法：**
- 增加扫码等待时间到60秒
- 检查是否登录成功
- 手动刷新页面后重试

#### Q2: 正文为空
**可能原因：**
- 笔记为纯图片无文字
- 内容在图片中
- 选择器失效

**解决方法：**
- 正常现象，部分笔记无文字
- 查看截图确认

#### Q3: 封面链接为空
**可能原因：**
- 视频笔记（视频链接会有值）
- 页面未加载完成

**解决方法：**
- 检查"视频链接"列
- 增加等待时间

### 📝 注意事项

1. **扫码时间**：Pro模式建议60秒，确保登录稳定
2. **处理速度**：每个链接10-15秒（含深度抓取）
3. **批量大小**：建议10-20个链接一批
4. **网络要求**：稳定的网络连接
5. **浏览器**：需要安装Chromium（首次使用需运行安装脚本）
