## Input handling

先识别任务模式,再决定分析框架:
- **内容诊断模式**:输入常见字段为 SKU、笔记大类、推广位置、小红星/小红盟、消费、CPC、CTR、CPUV、UV%、笔记类型、笔记链接。

- 若字段不完整,仍先做可判断部分,并明确说明缺失字段会影响什么结论。

## Core rules

- 不把不同分析维度混在一起。SKU、笔记大类、推广位置、合作模式(小红星/小红盟)默认分开看。
- 不只报结果,必须解释是"封面标题问题""正文承接问题""人群问题""竞价问题"还是"样本不足"。
- 批评必须配套修改方案。
- 样本不足时,不做激进调整。
- 有 JD/TM 双目标时,**优先保 CPUV,再看 CPC**。

## Mode A: 内容诊断

### A1. 先做数据清洗

识别并整理:
- 基础表现:消费、CTR、CPC、UV%、CPUV
- 业务上下文:SKU、笔记类型、推广位置、合作模式、流量场域
- 若用户给了双优/TOP 笔记,优先做对照分析

### A2. 用 4 象限诊断素材

#### 1) 高 CTR + 高 UV%:双优素材
结论方向:可复制、可放量。
重点提炼:
- 封面构图是否更聚焦
- 标题是否有明确 hook / 痛点 / 利益点
- 正文与内图是否有效承接首屏承诺
- 是否有更强原生感、真实性、场景感、情绪价值

#### 2) 高 CTR + 低 UV%:点击陷阱
结论方向:首屏会拉点击,但笔记内容承接差。
重点排查:
- 正文阅读体验差
- 内图质量弱或信息重复
- 缺少真实体验/前后对比/细节证据
- 话题跑偏,种草力不足
- 叙事结构混乱,没有从痛点走向解决方案

#### 3) 低 CTR + 高 UV%:精准潜力
结论方向:内容对,但封面标题拉新不足。
重点排查:
- 首图不够原生
- 标题没有 hook
- 痛点表达不够锋利
- 没把人群/场景/利益点说清楚

#### 4) 低 CTR + 低 UV%:垃圾素材
结论方向:点击和承接都弱,优先止损或重做。
重点排查:
- 内容切入跑偏
- 封面标题无吸引力
- 笔记质量和可信度都不足
- 产品卖点、人设、场景都不成立

### A3. 做元素归因

必须覆盖以下 **4 层 7 元素**(覆盖越多越好,缺什么就在优化建议里补什么):

#### 视觉层(封面/标题)
- **文案压字**:封面压字是否直击痛点/结果/利益点,还是堆砌卖点
- **切入痛点**:标题是否用具体场景或痛点开场,而非产品功能介绍

#### 内容层(正文/内图/视频前段)
- **情绪共鸣**:内容是否激发情绪(代入感、羡慕、焦虑解除、身份认同)
- **原生感**:整体风格是否像真实用户分享,而非广告感
- **真实性**:是否有真实体验细节、前后对比、成分解读、实拍证据

#### 转化层
- **利益点**:产品卖点是否用"我"的体验语言表达,而非成分堆砌
- **号召购买**:结尾是否有明确行为引导(进店/加购/送礼/收藏)

#### 用户心理层
判断内容更像在激发哪种动机:
- 进一步了解产品
- 收藏备用
- 购买/进店
- 节日送礼
- 自我提升/悦己/身份认同

> **元素缺一不可**:每个笔记至少覆盖 5/7 元素才算内容合格,缺什么就在优化建议里补什么。

### A3b. 行业对标参考(待投喂)

若用户提供了以下对标素材,优先做对照分析:
- **行业双优笔记**:CTR 和 UV% 均表现好的笔记(笔记链接 + 流量场域 + 笔记类型)
- **品类 TOP 笔记**:消耗大且 CPUV 优的笔记(笔记链接 + 流量场域 + 笔记类型)

对照分析维度:
1. 封面构图与压字方式的差异
2. 标题切入角度的差异(痛点型 vs 结果型 vs 场景型)
3. 正文叙事结构的差异(人设优先 vs 产品优先 vs 场景优先)
4. 元素覆盖完整度的差异

若用户未提供对标素材,在输出中注明"缺少行业对标,建议补充双优/TOP笔记作为参照"。

### A4. 输出动作建议

把建议写成可以直接 brief 内容团队的语句。

不要写:
- "提高点击率"
- "增强真实性"

要写成:
- "封面压字改成'熬夜脸又黄又糙?这套急救维稳我真用住了'"
- "标题从产品介绍改为痛点切入,例如'换季一泛红就烂脸的人,真的要看这篇'"
- "正文首段先讲使用场景,再补使用前后变化,不要一上来堆产品卖点"
- "内图增加1张近景肤质细节图 + 1张上脸前后对比图"

## Mode B: 账户调控

### B1. 先过滤白名单

默认不纳入调控计算（来源于数据表 `过期判断` + 设置表 `启用暂停`）：
- `过期判断 = 过期`：已过期计划，不参与调控
- `过期判断 = 无任务`：合作广场无任务状态，不参与调控
- `启用暂停 = 暂停`：已暂停计划，单独列入观察名单
- 计划创建时间为昨日：标记为观察期，不做大幅调整

上述计划列入"观察名单"，但不参与正式调控建议计算。

### B2. 目标优先级

若用户未另给目标,默认:
- 第一优先级:CPUV
- 第二优先级:CPC
- 第三优先级:消费稳定,不出现明显 GAP

冲突处理:
- CPUV 达标、CPC 超标:优先保 CPUV,只做小幅压价或控制预算节奏
- CPC 达标、CPUV 超标:判定为无效流量,必须收缩
- 两者都达标:优先放量
- 两者都超标:止损

### B3. 数据判断框架

优先用 7D、3D、1D 联合判断,建议权重:
- 过去 7 天:45%
- 过去 3 天:35%
- 过去 1 天:20%

若样本过小:
- 近 2 天新计划,且点击 < 50 或进店 < 20
- 标记为"观察期"
- 不做大幅调整,只监控 CPC 与基本起量能力

### B4. 四象限判断

基于对应 Content_Type 的目标值比较 CPC 与 CPUV:

- **A 优质放量**:CPC ≤ 目标,CPUV ≤ 目标
- **B 精准高价**:CPC > 目标,CPUV ≤ 目标
- **C 廉价无效**:CPC ≤ 目标,CPUV > 目标
- **D 劣质高危**:CPC > 目标,CPUV > 目标

### B5. 默认调整策略

#### A 优质放量
- 预算:上调 10%-20%
- 出价:维持或微调 +5%
- 逻辑:成本健康,优先抢量,但不要大幅破坏模型

#### B 精准高价
- 预算:维持或微调 +5%
- 出价:下调 5%-10%
- 逻辑:先压 CPC,同时尽量守住 CPUV

#### C 廉价无效
- 预算:下调 20%-50%
- 出价:下调 10%-20%
- 逻辑:点击便宜但不进店,属于无效流量,需明显收缩

#### D 劣质高危
- 预算:暂停或降到最低
- 出价:下调 20% 或暂停
- 逻辑:双超标,无继续放量价值

### B6. 调整约束

- 单次出价调整幅度不得超过 ±20%
- 预算调整必须考虑账户整体消耗缺口,不可只看单计划效率
- 若同一 Content_Type 需要预算迁移,优先从 C/D 象限挪给 A 象限

### B7. 辅助判断

#### 看 CTR
- CPC 高 + CTR 低:偏创意问题,降价同时标记"需优化素材"
- CPC 高 + CTR 高:偏竞争问题,可控幅调整,不要误杀

#### 看进店率(UV/Click)
- CPUV 高 + 进店率低:典型点击后流失,优先压价/缩量

#### 看渠道归因
- 若 JD/TM 差异很大,分别说明,不要合并成一个结论
